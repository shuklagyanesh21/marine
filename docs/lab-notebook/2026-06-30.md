# 2026-06-30

## Goals for today
- Standardize the raw genome layout into one canonical, deduplicated layer without touching `data/raw/`.

## What I did
- Implemented a new `marine_peptides.standardize` package plus thin wrapper `scripts/06_standardize_genomes.py`.
- Added a config-driven standardized layer under `data/interim/genomes/` with:
  - canonical semantic-prefix IDs (`ncbi_GCA_...`, `gorg_...`, `oceandna_...`)
  - one `index.tsv`
  - one `rosetta.tsv`
  - one `dedup_report.tsv`
  - symlinked FASTA view in `data/interim/genomes/fasta/`
- Ran the full content-hash standardization job across the current raw catalogs.

## What worked
- Input inventory: **127,033** source-instance genome rows.
- Unique on-disk files hashed: **55,234**.
- Final canonical genomes: **115,794**.
- Collapsed duplicate clusters: **10,544**.
- Standardized FASTA symlinks emitted: **50,632**.
- Output surfaces written to:
  - `data/interim/genomes/{index,rosetta,dedup_report}.tsv`
  - `data/processed/{genome_index,genome_rosetta}.tsv`
- Full run completed successfully in **88.5 min** with **0 warnings**.

## What failed / blockers
- The first full md5 run was effectively stalled by overly coarse multiprocessing chunking, so progress was not visible and throughput was poor. I stopped it, reduced chunk size, and re-ran successfully.
- A pre-hash validation run left **2,913** stale symlinks in `data/interim/genomes/fasta/`; I fixed the emitter to clear the standardized FASTA directory before writing and reconciled the current directory to the finished index.

## Decisions
- Keep `data/raw/` immutable; standardized layout lives in `data/interim/genomes/`.
- Preserve mixed compression (`.fna` and `.fna.gz`) in the standardized layer; do not decompress/recompress for now.
- Canonical winner priority: `ncbi > marref > mardb > gomc > gorg > oceandna`.
- Exact cross-source duplicates are collapsed by normalized sequence md5; near-duplicates remain a later dereplication step.
