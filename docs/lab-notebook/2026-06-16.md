# 2026-06-16

## Goals for today
- Build a marine genome **manifest** (what to download) from GTDB R202 metadata. No genome
  downloads yet — just accessions + provenance.

## What I did
- Downloaded GTDB R202 metadata/taxonomy/aux files into `data/metadata/gtdb/` (read-only).
- Implemented a two-tier marine-selection pipeline (config-driven, logic in the package):
  - `download/gtdb_metadata.py` — load+merge bac120/ar122 metadata, normalize accessions
    (`GB_`/`RS_` -> NCBI GCA/GCF).
  - `download/marine_filter.py` — whole-word include/exclude/host keyword classifier +
    lat/lon and depth parsers (pure, testable).
  - `download/biosample.py` — **Tier 2**: NCBI E-utilities `esearch`->`efetch(by UID)` to
    pull BioSample `env_*`/isolation/geo attributes; batched, rate-limited, cached/resumable.
  - `download/manifest.py` — merge tiers into the provenance manifest TSV.
  - Thin scripts `scripts/00_select_marine_candidates.py`, `01_enrich_marine_biosample.py`,
    `02_build_manifest.py`.

## What worked
- Tier 1 (offline GTDB keyword filter): **1,831** marine genomes (858 representatives);
  evidence dominated by seawater / ocean / marine sediment / coast / estuary.
- Tier 2 fetcher validated on a 320-sample mix of SAMN/SAMEA/SAMD: 320/320 resolved.

## What failed / blockers
- GTDB metadata `ncbi_isolation_source` is ~80% empty and `ncbi_lat_lon` ~95% empty, so
  keyword filtering alone has low recall -> motivated Tier 2 (BioSample ENVO lookup).
- `efetch -db biosample -id <accession>` only resolves NCBI-native **SAMN**; EBI **SAMEA**
  (~19.5k of the pool) and DDBJ **SAMD** return empty. Fix: `esearch` accession->UID first,
  then `efetch` by UID, back-mapping via the accessions embedded in the returned XML.
- The EDirect `efetch` perl wrapper hung in non-interactive shells; using raw HTTP instead.

## Decisions
- Marine scope = **broad** (seawater/coastal/deep-sea/hydrothermal/sediment/reef/...),
  host-associated genomes **tagged** (`is_host_associated`) not dropped; all keyword lists
  live in `config.yaml` so scope is tunable.
- Manifest includes **both representatives and cluster members** (`is_gtdb_representative`
  column) so the download step can subset either way.
- Tier 2 query pool = environmental genomes (MAG/SAG/env) with a BioSample not already
  flagged by Tier 1: ~54k BioSamples.

  ## Next steps

- Review precision of weaker keywords (`coast`, `lagoon`, `reef`) and the geo_loc-only hits.
- Implement the download stage (ncbi-datasets-cli) driven by the manifest accessions.
- (Optional Tier 3) integrate curated marine catalogs (MarRef/MarDB, OceanDNA, GORG).
