- Launched the full run under SLURM (`-profile slurm -resume`, concurrency 12); it completed
  **410** genomes and then **crashed the whole pipeline** on `ncbi_GCA_019894665v1`.
- Root-cause of the crash (chain):
  1. SmORFinder's bundled "DeepSmORFNET" prodigal **segfaults** (exit 139, core dumped) during
     the single-mode *training* phase ("Locating all potential starts and stops...").
  2. `run_prodigal` ignores prodigal's exit code, so `prodigal.gff` is left empty (one blank line).
  3. `filter_prodigal_small_genes` (`prodigal.py:188`) does `''.split('\t')` -> `['']` and then
     `line[2]` -> `IndexError`; `smorf single` exits 1.
  4. Our `errorStrategy` only retried exit codes `[137,140,143,247]`, so exit 1 -> `terminate`,
     killing all remaining ~112k tasks.
- Empirical crash rate (200 random genomes, bundled prodigal, deterministic sample):
  - single mode: **38/200 (19%)** crash (exit 134/139, core dumped)
  - meta mode (`-p meta`, no training phase): **1/200 (0.5%)** crash
  - => the crash is specific to single-mode training; meta mode avoids it.
- Fix applied: `modules/smorfinder.nf` `errorStrategy` now retries only transient resource codes
  and otherwise **`ignore`s** a failed genome so one bad assembly cannot terminate the batch;
  non-completed genomes are reported in the run manifest and retried on `-resume`.
- Noted (not the crash cause): the generated TSVs use CRLF (`csv` default `\r\n`); harmless to the
  pipeline (Groovy `readLines`/Python `csv` strip it) but trips raw `cut`/`awk` on the file.
- Open decision before relaunch: run everything in `meta` mode (recovers the ~19%) vs keep
  `single` + resilience (drops ~19% as failed) vs hybrid (single then re-run failures in meta).
- Validated meta recovery: `smorf meta` completed (exit 0, 3 predictions) on the exact genome
  (`ncbi_GCA_019894665v1`) that segfaulted in single mode.
- Decision: **hybrid**. Implemented:
  - `promote_failures_to_meta` + thin `scripts/13_promote_smorfinder_failures.py`: flip every
    `single` row lacking a valid `single/<id>/_SUCCESS.json` to `meta`; idempotent.
  - Right-sized `meta` resources to 2 CPU / 4 GB (small MAGs) so the recovery phase stays parallel.
  - Fixed `_write_tsv_atomic` to emit `\n` (was CRLF via csv default) and regenerated the input
    TSV (112,414 single rows, clean LF).
  - Added unit test for the promotion; full suite green (8 passed).
- Hybrid run flow (one chained command, resumable):
  `nextflow run smorfinder.nf -profile slurm -resume && python3 scripts/13_promote_smorfinder_failures.py && nextflow run smorfinder.nf -profile slurm -resume`
