## What I did
- Installed SmORFinder in a dedicated `smorfinder` conda env (Python 3.8); the PyPI
  package cannot install on Python 3.11/3.12 because it pins `tensorflow==2.3.1`.
- Added `env/smorfinder.yml` with `python=3.8` and `smorfinder==1.0.0`.
- Pinned `protobuf==3.20.3` in the SmORFinder env because TensorFlow 2.3.1 fails
  to import with modern protobuf 5.x.
- Documented the separate env in `README.md` and `AGENTS.md`; added `smorfinder` config
  block to `config/config.yaml` (output dir, conda env name).
- Left `pyproject.toml` unchanged — SmORFinder does not belong in the uv/Python 3.11 stack.

## Notes

- On this workstation, create the env under `/mnt` rather than `/home` because the
